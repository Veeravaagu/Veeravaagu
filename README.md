## Technical Depth

At the **systems level**, I work in C++17 — implementing storage engines, indexing structures, query execution models, and memory-constrained algorithms. I understand how software behaves when hardware constraints are real.

At the **backend level**, I design APIs and service layers in Python and TypeScript — building systems that are modular, testable, and production-ready. This includes real-time pipelines, authentication systems, and AI-integrated services.

At the **infrastructure level**, I containerize with Docker, deploy on cloud platforms, and build CI/CD-aligned workflows that make releases predictable and recoverable.

I use AI tooling — including LLM APIs and AI-assisted development — with applied engineering judgment. Generated outputs are treated as inputs to review, not final answers.

I design systems where AI is part of the architecture — not just an API call — with attention to state, failure modes, and long-term behavior.

## Engineering Philosophy

**On architecture:**
Separate what changes from what stays stable. Systems should be easy to modify without introducing risk.

**On debugging:**
Production failures are signals. Trace from the boundary inward, instrument before guessing, and fix root causes.

**On shipping:**
Working software matters — but correctness is non-negotiable. A system that silently fails is worse than one that fails loudly.

**On AI tooling:**
AI accelerates development, but only when paired with strong engineering judgment. Generated code is reviewed, validated, and treated like any other dependency.
